# Directus Mobile 0.14.0 — Better Relational Drafting

**0.14.0** focuses on one thing: making relation editing feel safer and easier while you are still drafting changes.

If you work with linked content (like related items, nested blocks, or repeatable sections), this update helps you move faster with fewer surprises.

## What is new in 0.14.0

The big improvement is a new **relational drafting setup** across:

- **M2O** (many-to-one)
- **M2M** (many-to-many)
- **O2M** (one-to-many)
- **M2A** (many-to-any)

In plain terms: you can now edit connected content more smoothly before saving, even when data is nested.

## Why this matters

Before, relation-heavy forms could become hard to manage when you mixed adding, editing, sorting, and removing items in the same flow.

In **0.14.0**, drafting behavior is more consistent and stable, so you get:

- **Cleaner draft state** while editing related records
- **Better nested editing** for complex relation structures
- **More reliable sorting** in relation lists
- **Safer updates** that reduce accidental overwrites
- **More predictable pick/add/edit/delete flows** for related items

### What you’ll notice while editing

- Editing related items feels less fragile, especially in longer forms.
- Reordering related content behaves more consistently.
- Existing related values are handled better during draft edits.
- Working with nested relational content is less error-prone.

This release is all about confidence: you can draft first, review changes, and save with fewer surprises.

---

If your projects rely heavily on relations, **0.14.0 is a worthwhile upgrade** for a smoother editing experience.
